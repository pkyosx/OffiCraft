進入 `server/` 時 nested-load。repo-wide 憲章 + 約定見 root `CLAUDE.md`;本檔記 server 專屬。棧:go1.26。

## 命名(root §10)
- **`server/ocserverd/`** — Go server daemon(**production server**;原 Python 實作已退役——歷史 = git tag `py-final`)。folder = `module ocserverd` = binary `ocserverd`。committed prebuilt 在 `bin/ocserverd`(CI step 1 同一 gate,自動被 `server/*/go.mod` 迴圈納入)。⚠️ 別跟 `bin/ocserver`(bash 的 server *installer*)混淆——`d` 是 daemon 本體。
- **`server/ocupdaterd/`** — 發佈/更新伺服器(**beta/GA 雙通道已入,owner 2026-07-14 拍板**):publisher 憑 publish token 上傳版本(`POST /api/publish`,server 端重算 sha256、不符拒收;**publish 一律先進 beta 通道**)、標 GA(`POST /api/promote`,body `{"version":...}`,冪等——已 GA 回原 stamp、不可改寫 promote 序;無 demote,出事 forward-fix 靠 promote 更新的版),受邀用戶憑 invite code 查最新(`GET /api/latest?channel=ga|beta`,**缺省 = ga**——不帶參數的舊客戶端自動落在穩定通道,決定卡 D1)/ 下載(`GET /api/binary?version=`,精確版號、任何通道皆可下載——channel 只決定 latest 指誰)/ 一鍵安裝(`GET /install.sh?code=&channel=`,缺省 pin GA latest)。資料面 = release 表一欄 `ga_at`(NULL=僅 beta;非 NULL=已 promote,GA latest = `ga_at DESC` 即 **GA 序 = promote 序**,與 publish 序解耦);舊 DB open 時 idempotent ALTER 補欄(`ensureGAColumn`,schema 仍非 goose)。發佈客戶端 = `cli/ocrelease`(publish/promote,憑證走 env)。`list-versions` 帶 CHANNEL 欄。folder = `module ocupdaterd` = binary `ocupdaterd`;committed prebuilt 在 `bin/ocupdaterd`。**刻意在 spec/openapi.json 與 conformance 之外**(那是 ocserverd 的凍結 wire,不是這隻 daemon 的);port 預設 8790、host 鎖 loopback。憑證 = 隨機 32-byte token 存 DB(sha256 hash 過、可撤銷,`credential` 一張通用表)——非 JWT,M1 從簡。**版號格式卡已定案(owner 2026-07-13):日期式 `vYYMMDD-NNNN`、publish 時 server 自動生成**——client 可不帶 version(server 取當日 prefix + 4 位當日流水 MAX+1,201 update-check 用的 code。**install.sh 預填已入(exit-beta)**:`GET /install.sh?code=` 生成的腳本在 download+sha256 驗證後,跑 `OC_UPDATER_URL=<baseURL(r)> OC_UPDATER_INVITE_CODE=<code> ./ocserverd set-updater` 把 updater 位址+邀請碼直落新 server 的 DB settings(`updater.url`/`updater.invite_code`)——base URL 用客戶端下載時的 face(`baseURL(r)`:Host + X-Forwarded-Proto,直連 loopback 或 tunnel 網域皆正確,無 canonical config 欄、與 download URL 同一推導);新 server 首次 serve 起站 snapshot 即載入,第一次 /api/version 更新檢查就生效,手動裝(不走 install.sh)路徑不受影響。⚠️ **部署位置設計卡仍待 owner 定案**;schema 用 idempotent `CREATE TABLE IF NOT EXISTS`(非 goose)——設計定案前不凍 migration 鏈。管理面兩張臉:本機 CLI 子命令(mint-publish-token / list-publish-tokens / revoke-publish-token / mint-invite / revoke-invite / list-invites / list-versions,直接操作 `oc-updater.toml` 解析出的 DB;store 層 list/revoke 泛化帶 kind 參數(`ListCredentials`/`RevokeCredential`)、kind gate 防跨撤——revoke-publish-token 永遠撤不到 invite,反向亦然)+ **管理 portal(`portal.go` + 內嵌單檔 `portal.html`,2026-07-14 入)**:`GET /portal/` 瀏覽器 UI + `/portal/api/*` JSON 面——版本列表(channel/發佈時間/git sha)、beta→GA 一鍵 promote(同 /api/promote 語意,帶確認、不可逆 forward-fix)、邀請碼管理(鑄碼 code 只在 201 回應出現一次、撤銷、列表)、**下游 fleet 監控**(一組 invite = 一台下游 OC:`GET /api/latest` 兼心跳,invite 行記 claim URL(claim token 存 DB、未 claim 前跨重啟穩定)、`POST /portal/api/set-password`(已設 409→claim 驗證 401→argon2id hash 落 DB、consume token、直接登入)、之後 `POST /portal/api/login` 換 7 天 session token(隨機 32-byte、DB 只存 sha256 hash,`portal_session` 表);其餘 portal 路由全 session-gated。argon2id 實作 = `password.go`(ocserverd 同檔 twin);portal/claim/session 憑證一律 hash 存放、絕不明文落 DB。
