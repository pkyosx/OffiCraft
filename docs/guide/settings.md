# 設定與參數

工作室的所有設定集中在一個地方：頂列的 **⚙️ 齒輪 › 設定**。這份帶你逐一看過五個子頁，以及最容易被找的 **參數調整**。

> **選**用哪個主題、語言、改密碼這類個人偏好在**頭像下拉 › 偏好設定**，不在這裡。唯一例外是**主題的「管理」**（新增／編輯／匯入／匯出）——它放在設定頁的「主題」子頁（見下）。簡單記：**偏好設定選主題，設定頁管理主題。**

---

## 設定的五個子頁

點頂列 **⚙️ 齒輪** 打開設定，左側有五個子頁：

| 子頁 | 放什麼 |
| --- | --- |
| **軟體更新（Software update）** | 目前版本、檢查與升級、接收 Beta、自動更新開關 |
| **角色誌（Role journal）** | 每個角色的定義、累積的學習經驗，以及三塊「全域情境」 |
| **任務手冊（Task manuals）** | 每一「類」任務的作業手冊（用途、要哪些欄位、SOP、負責誰） |
| **參數調整（Parameters）** | 登入有效期、自動換手門檻 |
| **主題（Theme）** | 管理主題：新增、編輯顏色與用語、匯入、匯出、刪除。詳見 [主題（外觀與用語）](theme.md) |

---

## 軟體更新

**升級不必重跑安裝腳本。** 這一頁就是你日常升級 OffiCraft 的地方：

- **目前版本** — 這台 server 現在跑的建置身分。
- **檢查更新** — 直接問 GitHub Releases 有沒有新版；有的話一鍵升級（下載 → sha256 驗證 → 原地抽換重啟）。
- **接收 Beta 版本** — 打開後也吃 GitHub 的 prerelease。**預設關閉。**
- **自動更新** — 打開後在背景自動升級，不用你手動按。**預設關閉。**

server 升級後，機器上的 warden 與 ocagent 也會跟著自我更新。完整的更新機制見 [架構與運作原理](architecture.md)；從頭安裝／升級／移除見 [安裝、升級與移除](install.md)。

## 角色誌

每個角色一塊，這裡看得到、改得動：

- **角色定義** — 這個職位是誰、負責什麼、遵守哪些規矩。你可編輯——改了它，這個角色的成員下次開機就讀到。
- **學習經驗（lessons）** — 這個角色累積下來的教訓，由成員結案時自動回寫，**同一角色的每個 session 共享**。
- **三塊「全域情境」**（每個成員開機時被注入的開機說明）：
  | 區塊 | 誰能改 | 是什麼 |
  | --- | --- | --- |
  | **系統互動** | 唯讀 | 系統給的通用開機內容（怎麼跟大家互動、用哪些工具） |
  | **使用者自訂** | **你可編輯** | 補上你這間工作室的規矩：你是誰、專案在做什麼、哪些事不能碰 |
  | **啟動程序** | 唯讀 | 開機當下照序做的步驟 |

> **「使用者自訂」是你最該花時間的一塊。** 把「你是誰、專案在做什麼、預算多少、哪些 repo 絕對不能動」寫進去，成員每次開機都讀得到，你就不必每次重講。這正是 OffiCraft 相對「開一堆終端機各跑一個 Claude Code」最省事的地方——詳見 [建議用法](best-practices.md)。

新增／刪除角色（＝多請或裁掉一位正職）的做法，見 [成員與外包](members.md)。

## 任務手冊

每一「類」任務的標準流程（SOP）：這是什麼任務、要哪些欄位（哪個是必填、哪個當識別鍵）、該怎麼做，外加累積的學習經驗，以及**指派給誰**（常駐成員或外包、跑幾份）。

手冊由你建一次、跨任務長期存在，而且**愈用愈準**——每張結案任務都把教訓回寫進同一本。手冊怎麼建、識別鍵怎麼挑，有一個完整例子在 [任務是怎麼運作的](tasks.md)。

> **手冊「由誰負責」只有你（owner）能設。** 那是治理面的設定，成員自己改不了。

## 參數調整

兩個伺服器層級的旋鈕：

| 參數 | 是什麼 | 可選 |
| --- | --- | --- |
| **登入有效期（Session length）** | 登入後多久需要重新輸入密碼 | 12 小時 / 24 小時 / 7 天 / 30 天 |
| **自動換手門檻（Auto-handover threshold）** | 一個成員的記憶用到這個比例，就自動把工作交接給新一手 | 40–90% |

**自動換手門檻**是這兩個裡比較值得理解的：成員的記憶（context）是有限的，用到門檻就會自動 checkpoint、收尾退場，server 原地重生一個新的它接著跑——你察覺不到接縫。門檻調低＝換手更勤、記憶更清爽但接手更頻繁；調高＝跑得更久才換。多數情況預設就好。（你也可以在成員面板按 **Refocus** 手動觸發一次換手，見 [成員與外包](members.md)。）

## 順帶一提：偏好設定在哪

**選主題、語言、改密碼**在**頭像下拉 › 偏好設定**：

- **主題** — 在這裡**選**要用哪個主題（內建的辦公室，或你已建立／匯入的自訂主題）。主題換的是顏色與用語、不動版面，也**不蓋過語言**。**新增／編輯／匯入／匯出**主題在設定頁的「主題」子頁——完整說明見 [主題（外觀與用語）](theme.md)。
- **語言** — 介面語言。
- **改密碼** — 換 owner 登入密碼。

頭像下拉裡還有**姓名**（可直接改）與**登出**。
