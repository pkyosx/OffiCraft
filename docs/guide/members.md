# 成員與外包

一間工作室裡幹活的是**成員**。這份講清楚兩種成員——**正職**與**外包**——差在哪、怎麼多請一個、怎麼管他們的上線與下線。

想先弄懂名詞，開 [名詞表](glossary.md)；想知道「什麼時候該外包」這種用法建議，看 [建議用法](best-practices.md)。

> 開箱時整間工作室只有兩個人：**你（owner），和 Mira**（你的助理，正職）。下面說的「多請一個」都是從這裡長出來的。

---

## 正職 vs 外包：一眼分辨

| | 正職（Staff） | 外包（Outsource） |
| --- | --- | --- |
| **身分** | 有名字、有**角色**、跨對話不忘的記憶 | 匿名，用**代號**稱呼（如 `O-7`），沒有角色記憶 |
| **在多久** | 常駐——你關掉瀏覽器它還在 | 臨時——綁一張任務，做完（或被終止）就退場 |
| **怎麼幹活** | 主 session 當**協調者**，重活交給 sub-agent 或發外包 | **自己下場做重活**（開發、測試、研究都自己來） |
| **記憶去哪** | 回寫進**角色學習經驗**（換人不換記憶） | context 用完即棄；經驗回寫進**任務手冊** |
| **數量** | 你一個角色一個角色地加 | 靠一個**全域同時雇用上限**兜底，滿額排隊 |

一句話：**正職是你養的同事，外包是你臨時發的一包活。** 需要長期深耕、要記住脈絡的事交正職；能獨立切出去、做完即結案的一次性活發外包。

辦公室側欄頂部就是這兩欄的分頁：**Staff（正職）** 與 **Outsource（外包）**，各自列出名單。

---

## 多請一位正職＝新增一個角色

正職成員的背後是一個**角色（role）**：角色定義「這個職位是誰、負責什麼、遵守哪些規矩」，成員 session 只是當下填這個位子的人。所以「請一位新正職」在 OffiCraft 裡，就是**新增一個角色定義**。

**兩條路都到同一個地方：**

- 辦公室側欄最下方點 **招攬新成員**（在 Staff 分頁時）——它直接帶你到角色誌的新增畫面。
- 或自己走 **設定 › 角色誌 › 新增角色定義**。

填一個角色名（例如 `repo-maintainer`、`researcher`）就建好了。角色一建立，這台工作室就多一個正職成員，warden 會在有 `claude` 與 `tmux` 的機器上把它叫醒。

> **角色定義與學習經驗都在 設定 › 角色誌 裡。** 每個角色一塊，你隨時改得動它的定義；它累積的學習經驗（lessons）也列在那裡。**換人不換記憶**——同一個角色換了新的 session，讀寫的還是同一份角色記憶。（怎麼寫好角色定義、把「你是誰／哪些事不能碰」沉澱進去，見 [建議用法](best-practices.md)。）

### 刪掉一個角色

在 **設定 › 角色誌** 對該角色按 **刪除**。要注意兩件事：

- **這是不可回復的**：該角色底下的成員、他們的對話與學習經驗會一起永久移除。
- **有成員線上時刪不掉**：得先讓它下線（見下方「上線與下線」），才能刪。

## 發外包：不用你一個一個開

外包不是你手動「請」進來的。它由**任務手冊**驅動：某一「類」任務在手冊裡被指定「交給外包做、一次跑幾份」，任務進來時系統就臨時開一個外包 worker 去做，做完自動退場解僱。

所以「發外包」你要碰的其實是**任務手冊**：

- 去 **設定 › 任務手冊**，把某類任務的負責對象設成外包，並填 **Hire count（雇用份數）**——同一類要並行跑幾份。
- 誰負責哪本手冊、跑幾份、在哪台機器，**只有你（owner）能設**——那是治理面的設定，成員自己改不了。

任務手冊怎麼建、識別鍵怎麼挑，見 [任務是怎麼運作的](tasks.md)。

### 全域同時雇用上限

外包開太多不會無限燒錢——有一個**全域同時雇用上限**兜底：**滿額就排隊，有名額自動接**，不用你逐張催。

在辦公室側欄切到 **Outsource（外包）** 分頁，點最下方 **招攬新成員** 會打開 **外包上限設定**：

| 設定 | 意思 |
| --- | --- |
| 一個數字（**最多雇用 N**） | 同時最多 N 個外包在跑，滿了排隊 |
| **無限**（∞） | 不設上限 |
| **0** | **暫停指派**——現有的跑完就不再開新的 |

分頁標題下的小字會顯示目前人數與上限（例如「3 人 · 上限 5」）。

### 外包名單上看得到什麼

每一列顯示外包的**代號**、**任務類型**、以及它**現在正在做的那張任務的標題**（沒有進行中任務時顯示「無當前任務」）。點一列就開跟這個 worker 的聊天（標題是「外包 · 代號」）；點它的頭像則開**外包詳情**。外包是匿名的，所以名單上不寫模型、不寫識別鍵——代號本身已經隱含了它用哪個模型。

## 上線與下線

成員在不在線，只由名字旁那顆 **presence 圓點**表示（顏色是唯一訊號）：

| 圓點 | 意思 |
| --- | --- |
| **Offline（離線）** | 沒在跑 |
| **Waking（喚醒中）** | 剛被叫醒、還在開機 |
| **Online（線上）** | 已就緒，可以對話 |
| **Stopping（停止中）** | 正在收尾退場 |
| **Stopped（已停止）** | 已停下 |

「線上」在 OffiCraft 裡有一個很乾淨的定義：**成員此刻有沒有持著那條到 server 的長連線**——持著就是線上，斷了就是離線，沒有心跳、沒有 TTL。背後的道理見 [架構與運作原理](architecture.md)。

**你能做的幾個動作**（成員詳情面板 / 外包詳情裡）：

- **Wake（喚醒）** — 把一個離線 / 已停止的成員叫回線上。就算它現在離線，你也可以先在聊天框留言，**訊息會排隊，它上線就讀到**。
- **Refocus（重新聚焦）** — 線上時可用。按下後成員會壓縮記憶、收尾這一手 session、重生成新的一手接著跑。**任務不會因此被關掉**，只是換了一手更清爽的記憶。（這是你主動觸發一次「換手」；系統也會在記憶快滿時自動換手，門檻在 設定 › 參數調整 調，見 [設定與參數](settings.md)。）
- **Stop（停止）** — 讓成員正常收尾後停下。
- **Force stop（強制停止）** — 立刻砍掉 session、跳過正常收尾。**進行中還沒存的工作會遺失**，是最後手段。

> **EFFORT · Thinking（思考強度）**〔正職面板〕/ **Effort（投入度）**〔外包詳情〕— 這個成員的推理強度（低／中／高）。改了之後**下次喚醒或換手時生效**（工作進行中改的話，已指派的下次啟動才套用）。

畫面上每個欄位更完整的對照，見 [介面說明](interface.md) 的「成員與外包」一節。
