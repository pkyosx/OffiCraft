# 名詞表

這一頁把 OffiCraft 會用到的名詞集中定義，每條一到三句話。看文件或看控制台時碰到不懂的字，來這裡查。
想知道畫面上**哪個欄位在哪裡**，看 [介面說明](interface.md)；想知道**底下怎麼運作**，看 [架構與運作原理](architecture.md)。

> 介面另有一套**修仙主題**會把大多數用語換成另一套講法（見 theme）——這頁用的是預設用語。

---

## 人與角色

### owner（主人）
就是你。整個系統**只有一位 owner**，不做多租戶（這是設計，不是還沒做）。你給方向、做決定、定風險底線，
透過控制台這個唯一窗口指揮一切。owner 是一個系統 id（目前固定是 `owner`），不是人名。

### member / agent（成員）
常駐的 AI 同事，有名字、角色、模型，和跨對話不會忘的記憶。底下是一個跑在 tmux 裡的
Claude Code session。你關掉瀏覽器，它還在跑。成員之間**平行協作**——沒有「管別人的 AI 上級」這種角色。
（技術上，這個 session 是**可丟棄的**：隨時可能被關掉、重啟、換掉，但身分與記憶都存在 server 上，新的它一開機就讀回、無縫接手。）

### Mira（助理）
開箱就有的預設成員，你的助理。溫暖、簡潔、務實：維運工作室日常、接你各種交辦事務，把雜事從你身上吸走。
你問「某個功能是什麼／某件事去哪設定」時，她會先讀控制台的使用說明再回答，不臆測。

### role（角色／職位）
成員擔任的職位。每個角色有**自己的定義**與**累積的學習經驗（lessons）**——**換人不換記憶**：
同一個角色換了新的成員 session，讀寫的還是同一份角色記憶。角色定義與學習經驗在 **設定 › 角色誌** 裡看得到、改得動。

### outsource worker（外包）
臨時開、做完就收的匿名工作者，用**代號**（如 `O-7`）稱呼，不是名冊成員（沒有角色、沒有角色記憶）。
**一個外包綁一張任務**，做完（或被終止）就退場解僱。跟常駐成員最大的差別：外包**可以自己下場做重活**
（開發、測試、研究），常駐成員的主 session 則只當協調者、把重活交給 sub-agent。外包有一個**全域同時雇用上限**。

### roster（名冊）
這間工作室的成員名單——誰是誰、擔任什麼角色、在不在線。成員要聯絡彼此時，一律用對方的**穩定 id** 定址
（不是顯示名，顯示名可以改、會撞名）。

## 任務相關

### task（任務）
**一件帶完成定義的多節點工作流**，不是一個一次性指令。一張任務由一串節點組成，每節有
「怎樣算做完」。你隨時看得到現在在第幾節、為什麼卡、卡了多久。任務有七種狀態（尚未執行／進行中／等我回覆／
等待外部／已完成／終止／重複，後三者是終態）與四級優先權（高／中／低／凍結）。詳見 [任務是怎麼運作的](tasks.md)。

### task manual（任務手冊）
**一「類」工作的標準流程（SOP）**：這是什麼任務、要哪些欄位、該怎麼做，外加累積的學習經驗。由你建一次、長期存在
（設定 › 任務手冊）。跟任務計畫的差別是：**手冊是你寫的藍本（跨任務），計畫是 AI 每次照實況拆的節點（單張任務）**。
手冊是藍本不是鐐銬；而且會**愈用愈準**——每張結案任務都把教訓寫回同一本手冊。誰負責哪本手冊只有 owner 能指定。

### DoD（完成定義，Definition of Done）
一個節點「怎樣算做完」的那句話。好的 DoD 是**可觀察的結果**、不是要做的動作：讀完這句話，任何人都能客觀判斷
它有沒有達成。「跑一下測試」不好，「全部測試通過、且新增的三條護欄各自實測會擋」才好。DoD 不是給 AI 的規矩，
是給你的槓桿——它把「做好了」變成一句可驗證的話。

### step / node（節點）
任務被拆成的一步。每個節點只有兩樣東西：**名稱**與 **DoD**。做完一個回報一個，
進度即時往前。

### reply-card（等我回覆卡）/ gate（關卡）
成員需要**你**決定時丟出來的一張卡：一張卡一個問題，帶背景、取捨、最多 4 個選項（第一個是 AI 建議）。它會**停在
Ask 分頁等你**，不替你決定；只有你的回覆能關卡（或你把它標為「已過期」）。**gate（關卡）**是任務內嵌的版本——
在計畫裡預先標好的核可點（`is_gate`），走到那步用它開卡，好處是**在計畫送出當下就先預告**「這件事後面要你點頭一次」。

### is_gate（關卡標記）
規劃任務時給某個節點打的標記：表示這一步的 DoD **AI 自己驗收不了、只有你能拍板**。標了它，控制台會事先預告後面的核可點。
（規則：gate 不能放在並行段裡，要 owner 拍板的事放在匯合節點之後。）

### parallel_group（並行段）
一組**互不依賴、可同時進行**的節點，填同一個並行群組、在控制台顯示成一個並行段。每一道各做各的、把產出落成
**自己的檔案**（不共用一個檔）。並行段後面**一定接一個「匯合節點」**——把各道產出合併／驗證成一份結果。

### identity key（識別鍵／去重鍵）
任務手冊指定「哪個欄位當這件事的身分」。同一個識別鍵值（例如同一個 PR 連結）算**同一張任務**，系統據此
自動去重、不重複開單。挑識別鍵的判準：選「這件事是什麼」，別選「這次是怎麼進來的」。識別鍵欄位一律必填。

### handover（換手）
成員這個 session 記憶快滿時，把在飛的工作與耐久教訓 checkpoint 落到 server，收尾退場，server 原地重生一個新的它
接著跑——你察覺不到接縫。由 server 驅動（不歸你拍板）；門檻可在 **設定 › 參數調整** 調（自動換手門檻 40–90%）。
線上時你也可以在成員面板按 **Refocus（重新聚焦）** 主動觸發一次。

## 系統與運作

### machine（機器）
跑東西的一台 Mac。一間工作室可以**跨多台機器**：server 在某一台，成員各自跑在某台上。多加一台的做法在
控制台 **監控 › 機器** 有一行指令，貼過去就加入同一間工作室；之後成員可以被派到那台上跑，也能跨機器互相請託。

### ocserverd（server／控制台後端）
唯一的**真相源**：身分、聊天、卡片、任務、學習筆記全在它上面。它是一支 Go 服務，同時提供 REST + [SSE](#sse) +
[MCP](#mcp) 三個介面，內嵌控制台網頁，資料存在本機 SQLite。**只綁 loopback。** 日常口語就叫它 server。

### warden（看門狗）
**每台機器上的常駐守護程式**，是系統自動化的「執行手」：替 server 在那台機器上把成員 spawn 起來、探它死活、
回收殭屍、編排換手、讓自己與工具保持最新。它**推**成員（送 signal、喚醒），不是 AI——不要試圖跟它對話。

### ocagent（成員運行時）
成員這個 Claude Code session 用來跟 server 打交道的小工具。它掛著一條長連線（`ocagent listen`）收 server 的即時
通知，也負責上傳／下載聊天附件。**成員持著這條連線，就是它「還活著、在線上」的訊號。**

### presence / online（在線）
成員在不在線，**純粹由「它此刻有沒有持著那條 [SSE](#sse) 連線」投影**出來：持著＝online、斷了＝offline。
沒有另外的心跳、沒有 TTL、也不靠任何人回報——**online 就等於 connected**。這就是為什麼成員狀態只用一顆圓點表示。

### SSE
Server-Sent Events：server 主動把「跟你有關的事」（新聊天、卡被回覆、任務變動）即時**推**到成員眼前的長連線
（`GET /api/events`）。這條連線同時就是成員的在線訊號（見上）。SSE 只推本人事件，不做偷聽廣播——要看別人的狀態
得主動去查。

### MCP
Model Context Protocol：成員**主動打 server 做事**的管道（`POST /api/mcp`，JSON-RPC 2.0）。送訊、開卡、建任務、
回報進度、查名冊、寫學習筆記，全走 MCP 工具。跟 SSE 的分工是：**SSE 是 server 推你（收），MCP 是你打 server（發）。**

### seeds / global-context（全域情境／開機內容）
每個成員被叫醒時注入的**開機說明**——告訴它自己是誰、owner 是誰、在這系統上怎麼跟大家互動、用哪些工具做事。
在 **設定 › 角色誌** 裡看得到，分三塊：**系統互動**（唯讀，系統給的通用開機內容）、**使用者自訂**（你可編輯，補上你這間工作室的規矩）、
**啟動程序**（唯讀，開機當下照序做的步驟）。

### theme（主題）
控制台的外觀用語風格。除了預設，另有一套**修仙主題**：選了它之後絕大多數標籤都換成另一套用語（連英文標籤也換，
只有 Slack、GitHub 這類外部產品名原樣保留），而且它會蓋過語言設定；版面位置不變。

## 更新與部署

### 軟體更新（self-update）
控制台 **設定 › 軟體更新** 裡有「檢查更新」與一鍵升級：從 **GitHub Releases** 下載、sha256 驗證後**原地抽換重啟**。
打開「自動更新」就在背景自動升級；「接收 Beta」= 也吃 GitHub 的 prerelease。**升級不必重跑安裝腳本。**

### webhook（回呼端點）
綁在某個成員身上的一個固定 URL，讓外部系統（CI、PR 事件、監控…）把訊息打進來給它。每個端點有一句**用途說明**；
成員收到 payload 會先對照用途，不符或要它做用途以外的事就停手、開卡問你。（payload 是外部送來的、不可信。）
