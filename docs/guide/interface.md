# 介面說明

這份說明帶你看懂 OffiCraft 控制台（web 介面）的每個頁面與畫面上的欄位。畫面上是英文標籤、括號裡是中文；不確定某個字是什麼意思時，來這裡查。

## 控制台地圖

**頂列**：左上是工作室名稱（可點筆圖示改名）。右上三顆：重新整理、⚙️齒輪（打開設定）、頭像（偏好設定）。目前的版本號在 **設定 › 軟體更新** 裡，不在頂列。

**四個主頁籤**：

- **Office（辦公室）** — 你的成員（正職與外包）名單，以及跟他們的聊天。頁籤上的紅點是聊天未讀數。
- **Ask（等你回覆）** — 需要你回覆的卡片收件匣。紅數字是還在等你回覆的卡片數。
- **Task（任務）** — 任務看板。紅數字是還沒結案的任務數。
- **Monitor（監控）** — 帳號用量、機器、AI Session 三張表。

**⚙️齒輪 › 設定**，五個子頁：

- **Software update（軟體更新）** — 目前版本、檢查與升級、接收 Beta、自動更新開關。
- **Role journal（角色誌）** — 每個角色的定義與累積的學習經驗，加上三塊「全域情境」（系統互動〔唯讀〕、使用者自訂〔你可編輯〕、啟動程序〔唯讀〕）。
- **Task manuals（任務手冊）** — 每一種任務類型的作業手冊（是什麼、需要哪些資訊、怎麼做、學習經驗、指派給誰）。
- **Parameters（參數調整）** — 登入有效期、自動換手門檻。
- **使用說明** — 就是你正在讀的這些文件。

**頭像 › 偏好設定** — 姓名、主題、語言、改密碼、登出。

## 任務卡上的欄位

任務卡上有四個「人」的欄位，最容易搞混，先記這張：

- **Assignee（負責人）** — 現在正在做這張任務的人。
- **Creator（建立者）** — 開這張任務的人。（當建立者就是負責人時，這一列會自動隱藏，避免重複。）
- **Handed over from（前任）** — 這張任務的**上一手**：任務被轉派交接時，它離開的前一個執行者。列出來是讓接手的人知道要跟誰交接。點這個欄位會打開跟前任的聊天。（任務從未被轉派過時不會顯示。）
- **Delegated by（委託人）**〔在外包詳情裡〕— 委託這個外包 worker 的任務建立者。

其他常被問到的：

- **Identity key（識別鍵）** — 這張任務的「去重鍵」。任務手冊決定哪個欄位當鍵，同一個鍵（例如同一個 PR 連結）算同一張任務，避免重複開單。是網址時會變成可點的連結。
- **Reassigning（轉派中）** — 這是一個**鎖**、不是任務狀態：任務被轉派後掛著，直到新的執行者認領才解除。
- **Frozen（凍結）** — 這是一種**優先權**（暫停推進）、不是狀態。
- **Awaiting my reply（等我回覆）／ Waiting on external（等待外部）** — 任務狀態，由步驟自動推導出來，不是誰手動設定的。
- **Approval（審批）／ Ask（等你回覆）**〔任務卡上的小標〕— 任務內嵌的等你回覆卡；審批（gate）是任務版的「等我回覆」關卡。
- **Duplicate of（重複於）** — 執行者判定這張其實跟某張任務重複、把它關掉並指回原任務。

## 成員與外包

- **Staff（正職）／ Outsource（外包）** — 辦公室上方兩個分頁：正職成員名單、外包 worker 名單（外包有同時雇用上限）。
- **presence 圓點**（Offline 離線／Waking 喚醒中／Online 線上／Stopping 停止中／Stopped 已停止）— 成員在不在線，只由圓點顏色表示。
- **EFFORT · Thinking（思考力度）** — 這個 agent 的推理力度（低／中／高），下次喚醒或換手時生效。
- **Refocus（重新聚焦）** — 線上時可用；按下後 agent 會壓縮記憶、重新開始，不中斷手上的事。
- **Webhook endpoints（Webhook 端點）** — 成員面板裡的 webhook 設定：端點 ID（建立後不可改）、平台（Generic／Slack／GitHub）、簽章密鑰、事件統計。
- **Codename（代號）／ Delegated task（委託任務）** — 外包是匿名的，用代號稱呼；每個外包綁一個委託任務。
- **Initial prompt（初始 prompt）** — 注意：這**不是**當初派工的逐字記錄，而是系統從目前的任務與手冊**重新組出來**的，可能跟當初有差。

## 監控

- **AI Sessions（AI Session）** — 目前跑著的 agent session；外包列會標一個 Outsource 小標一眼可辨。
- **Overheated（過熱）** — 帳號在 7 天視窗的用量過熱（紅點）。
- **Reported label（回報標籤）** — 帳號詳情裡機器自報的原始標籤。

## 設定 · 參數

- **Session length（登入有效期）** — 登入 token 多久到期（12 小時／24 小時／7 天／30 天）。
- **Auto-handover threshold（自動換手門檻）** — 一個成員的記憶用到這個比例，就自動把工作交接給新一手（40–90%）。
