# 為什麼是 OffiCraft

**Craft your own AI office.** OffiCraft 是一間跑在你自己 Mac 上的 AI 工作室：你僱幾位常駐的
AI 成員，把事情整件交給他們，在一個網頁控制台裡看他們做到哪、在他們需要你點頭時回一句。
跑的就是你機器上那個 Claude Code。

這份文件回答一件事：**你已經有 Claude Code 了，為什麼還要多養一個常駐服務？**

---

## 這適合你，如果…

- ✅ 你同時開著好幾個終端機各跑一個 Claude Code，已經搞不清楚誰在做什麼
- ✅ 你受夠每次都要重講一遍你是誰、專案在做什麼、哪些事不能碰
- ✅ 你已經把 Claude Code 調好了（skill、plugin、MCP），不想為了換一套工具重來一次
- ✅ 你手上不只一台 Mac，想讓它們變成同一間公司的不同辦公室
- ✅ 你要 AI 在該問你的時候停下來問，而不是自己決定完才告訴你
- ✅ 你要全部跑在自己機器上，資料一步都不出去

## 它不是什麼

|  |  |
| --- | --- |
| **不是 agent framework。** | 我們不教你怎麼寫 agent。我們管的是一群 agent 怎麼一起上班。 |
| **不是雲端服務。** | 沒有註冊、沒有我們的伺服器。全部在你的 Mac 上。 |
| **不是團隊協作工具。** | 只有一位 owner，不做多租戶。這是設計，不是還沒做。 |
| **不是聊天機器人。** | 成員有職位、有記憶、有正在進行的任務——不是一個對話框。 |

## 一天長什麼樣

| | 你 | 它 |
| --- | --- | --- |
| **早上** | 丟一句「把這個 repo 的 README 重寫」 | 拆成幾個節點，每節寫下「怎樣算做完」，開始推 |
| **中午** | 你去忙別的 | 做到第三節卡住了——它停下來，把問題整理成一張卡放進 Ask |
| **晚上** | 打開 Ask，看到問題與選項，點一下 | 接著把剩下的做完，成果釘回任務卡上 |

## 你會得到什麼

|  |  |
| --- | --- |
| **常駐的成員** | 有名字、職位、模型，和跨對話不會忘的記憶。你關掉瀏覽器，他們還在跑。 |
| **任務，不是指令** | 一件事拆成多個節點，每節寫明完成定義（DoD）。你隨時看得到卡在哪一節、為什麼卡。 |
| **Ask：該你決定的才給你** | 需要你拍板的事會被整理成一張帶選項的卡，等你回一句。不需要你盯著螢幕。 |
| **你原本的 Claude Code** | 成員跑的就是你機器上那個 `claude`。你串好的 skill / plugin / MCP 原封不動全都在，不用為了這套工具重配一次。 |
| **多台機器一間公司** | 控制台給你一行指令，貼到另一台 Mac 上就加入了。成員可以被派到那台上跑。 |
| **成員之間看得到彼此** | 每位成員知道同事是誰、負責什麼。需要別台機器上的同事幫忙時直接開口，不用你轉達。 |
| **不只是文字** | 截圖、mockup、設計稿、報告直接在聊天與任務卡上傳與預覽。你圈一張圖給它看，它做完把成品掛回卡上。 |
| **手機也能推進** | 控制台為手機做過設計，通勤路上回一張 Ask 卡、看今天誰在忙什麼都可以。（控制台只綁 loopback，要從外面連得自己開一條 tunnel。） |
| **用量一眼看完** | 誰在哪台機器、用哪個模型、花了多少、5 小時與 7 天的額度還剩多少——同一頁。額度快見底時成員會自己放慢，不用你去踩剎車。 |

## 它怎麼運作

```
你 (owner)
 └─ 成員 (member) ── 一個跑在 tmux 裡的 Claude Code session
      └─ 任務 (task) ── 拆成多個節點，每節有明確的完成定義 (DoD)
           └─ 需要你點頭 ──> 一張卡進 Ask ──> 你回一句 ──> 繼續
```

| 名詞 | 是什麼 |
| --- | --- |
| **owner** | 你。整個系統只有一位，不做多租戶。 |
| **成員（member）** | 常駐的 AI 同事。底下是一個跑在 tmux 裡的 Claude Code session。 |
| **角色（role）** | 成員擔任的職位。每個角色有自己的定義與持久記憶——換人不換記憶。 |
| **任務（task）** | 一件事，拆成帶完成定義的多節點工作流。 |
| **Ask（等我回覆卡片）** | 成員需要你決定時丟出來的東西。它會停在那裡等，不會替你決定。 |
| **控制台（Console）** | 你指揮這一切的網頁。聊天、任務、成員、監控都在裡面。 |
| **server** | 唯一的真相源，只綁 loopback。 |
| **warden** | 每台機器上的常駐守護程式，負責把成員起起來、收回去。 |

**server 和 warden 只是水電，產品是你僱的那些人。**

開箱時整間工作室只有兩個人：**你，和 Mira**（你的助理）。

---

## 誠實的邊界

- 🔴 **成員的 Claude Code 是用 `--dangerously-skip-permissions` 跑的。** 這是刻意的：成員是無人看管的背景 session，跳出權限確認框沒有人會按，只會讓它永遠卡住。
  代價要講明白：**成員在你的機器上有很大的行動自由**，護欄靠的是它讀到的工作守則、以及「需要你點頭的事會停下來開一張 Ask 卡」這個機制，不是靠一道系統權限牆。
  這件事你要接受了再裝。（同理，`AskUserQuestion` 這個會彈互動選單的內建工具是被關掉的——headless 環境沒有人能按它。）
- **只支援 macOS Apple Silicon。** 安裝腳本會在其他平台直接拒絕。
- **這台機器得有已登入的 `claude` 與 `tmux`。** 解析不到 `claude` 時 warden 會直接拒絕安裝，而不是裝一個永遠起不了成員的 warden。
- **成員用的是你自己的 Claude 額度。** 這是「跑在你機器上」的另一面：帳單也是你的。
